# 总览

P2P 网络要求通信双方都能主动发起访问，但是 NAT 的存在，却阻断了这种主动访问，导致 P2P 应用无法正常运行。我们需要在通信双方之间建立一条可以穿越 NAT 设备的数据通道，实现 P2P 通信。ICE 框架整合了多个协议实现了这一目的。

首先双方都需要知道对方的 ip 地址和端口，需要TURN 服务器的协助

假设 R 拿到了 L 的 ip 地址和端口，由于有NAT的存在，可能也无法通信

所以 L 需要通过STUN 服务器知道自己的 NAT 地址，再告诉 R

但由于一些特殊 NAT 类型的存在，即使双方告知了对方 NAT 地址也无法互通，这时候就需要TURN/Relay 服务器起到中继的作用，帮忙传递消息，以实现互通
